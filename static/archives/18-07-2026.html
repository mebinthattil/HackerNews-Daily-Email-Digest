The Zilog Z80 has turned 50

179 points • by st_goliath • 28 comments
Summary
The Zilog Z80, launched in July 1976, marked the 50‑year anniversary of a processor that became a de facto standard for 8‑bit microcomputers, enabling CP/M and Microsoft BASIC and spawning clones such as the Sharp LR35902 (Game Boy). It evolved from Intel’s 8008 and 8080: the 8008’s 7‑register, 14‑bit address, internal return‑stack design gave way to the 8080’s external 16‑bit stack, 64 KB address space, and expanded I/O. The Z80 retained binary compatibility with the 8080 while adding IX/IY index registers, bank‑switched register pairs, three interrupt modes, bit‑manipulation, BCD arithmetic, and block‑transfer instructions (e.g., LDIR). Electrically, it required only a single +5 V supply and one clock, exposing clear MREQ/IORQ, RD/WR, and M1 signals that simplified system design and DRAM refresh. Developed by Federico Faggin and Masatoshi Shima at Zilog (funded by Exxon), the chip cost ≈ $400 k to prototype and was second‑sourced by Mostek. Zilog later produced the 16‑bit Z8000 (1979) and, after multiple ownership changes, discontinued the Z80 in June 2024.

Summary
Security advisory for TP‑Link Kasa Spot EC71 (firmware 2.3.26) identifies three critical vulnerabilities, all patched in firmware 2.4.1 (CVE‑2026‑9770, CVE‑2026‑13230): - **Hard‑coded RSA keys**: Two fleet‑wide RSA key/certificate pairs (1024‑bit SHA1‑signed, expired; 2048‑bit SHA256‑signed, valid to 2031) are stored in SquashFS layers and can be extracted from SPI flash. The active private key is identical on every device, enabling decryption or forging of TLS traffic if exploited. - **Insecure credential storage**: User TP‑Link ID passwords are saved as unsalted MD5 hashes in `config/account`, with the email address stored in plaintext on the JFFS2 overlay. The hash is trivially cracked, allowing cross‑domain account takeover across all TP‑Link services (e.g., smart locks, mesh routers). - **Unauthenticated GPS exposure**: An unauthenticated UDP request to port 9999 (`{"system":{"get_sysinfo":{}}}`) returns JSON containing precise GPS coordinates, hardware IDs, MAC addresses, and firmware version, protected only by a trivial XOR. Coordinates are stored permanently from account creation and are broadcast regardless of geofencing settings. The advisory documents a secondary‑market attack: a second‑hand device can reveal the prior owner’s location and credentials via SPI flash extraction and the UDP query, enabling full account compromise without prior network access. Firmware 2.4.1 removes the fleet‑wide RSA files, encrypts credential storage, and disables the GPS response.
